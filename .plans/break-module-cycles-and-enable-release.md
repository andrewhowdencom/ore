# Plan: Break Module Cycles and Enable External Release

## Objective
Restructure the `ore` Go multi-module repository to eliminate circular module dependencies between the root module and its sub-modules. This enables external consumers to `go get` sub-modules without hitting `replace`-directive failures or dependency resolution errors. The root module becomes a pure core framework; shared parent packages under `x/` become standalone modules; examples and the docgen tool leave the root module.

## Context

### Current Module Layout
- **Root module** (`github.com/andrewhowdencom/ore`): Contains `artifact/`, `cognitive/`, `loop/`, `provider/`, `session/`, `state/`, `thread/`, `agent/`, `x/guardrails/`, `x/systemprompt/`, plus `examples/` and `cmd/docgen/` (no `go.mod` of their own).
- **Sub-modules** (10 total): `x/conduit/http/`, `x/conduit/slack/`, `x/conduit/telegram/`, `x/conduit/tui/`, `x/provider/openai/`, `x/tool/bash/`, `x/tool/calculator/`, `x/tool/filesystem/`, `x/tool/mcp/`.
- **Parent packages without `go.mod`**: `x/conduit/` (shared conduit interfaces), `x/tool/` (shared tool interfaces). These are currently resolved **through the root module**, forcing all sub-module consumers to transitively depend on the entire root.
- **Cycle source**: `examples/` and `cmd/docgen/` import sub-modules (e.g., `x/provider/openai`, `x/conduit/tui`). Because they lack `go.mod`, they are part of the root module, creating a bidirectional dependency (root → sub-modules → root).
- **Replace directives**: Every sub-module has `replace github.com/andrewhowdencom/ore => ../../..`. The root `go.mod` has 6 `replace` directives for sub-modules. `replace` is ignored by `go get`, so external consumers cannot resolve versions.

### Files Discovered During Discovery
- `go.mod` — root module, contains 6 `replace` directives and `require`s for sub-modules
- `x/provider/openai/go.mod`, `x/conduit/http/go.mod`, `x/conduit/tui/go.mod`, `x/conduit/slack/go.mod`, `x/conduit/telegram/go.mod`, `x/tool/bash/go.mod`, `x/tool/calculator/go.mod`, `x/tool/filesystem/go.mod`, `x/tool/mcp/go.mod` — all contain `replace` for root
- `cmd/docgen/main.go`, `cmd/docgen/main_test.go` — docgen tool to be deleted
- `examples/*/main.go` — calculator, filesystem, http-chat, single-turn-cli, tui-chat
- `x/conduit/conduit.go` — leaf package, imports only `context`
- `x/tool/tool.go`, `x/tool/registry.go` — imports `context` and `github.com/andrewhowdencom/ore/provider`
- `docs/conduit-capabilities.md` — generated by docgen, to be replaced or deleted

### Architectural Decision
`x/conduit/` and `x/tool/` (parent packages) will become standalone modules. They are imported by multiple sub-modules, have minimal dependencies (`x/conduit/` is stdlib-only; `x/tool/` depends only on root `provider/`), and keeping them in the root module forces unnecessary transitive bloat on sub-module consumers. `x/guardrails/` and `x/systemprompt/` remain in the root module — they are not imported by any sub-modules.

## Requirements
1. The root module must not import any sub-modules (no `require` or `replace` for `x/` paths).
2. Sub-modules must not use `replace` directives pointing to the root module.
3. External consumers must be able to `go get github.com/andrewhowdencom/ore/x/provider/openai` (and similar) without errors.
4. Local development must work via `go.work` without `replace` directives in published `go.mod` files.
5. `cmd/docgen` is deleted; `docs/conduit-capabilities.md` is removed or manually maintained.
6. All existing tests must continue to pass (`go test -race ./...`).

## Task Breakdown

### Task 1: Create `x/conduit` Standalone Module
- **Goal**: Extract the shared conduit interface package (`x/conduit/`) into its own Go module so conduit sub-modules depend on it directly instead of through the root module.
- **Dependencies**: None.
- **Files Affected**: None (reads existing files to determine imports).
- **New Files**: `x/conduit/go.mod`
- **Interfaces**: New module path: `github.com/andrewhowdencom/ore/x/conduit`
- **Validation**: `cd x/conduit && go mod tidy && go build ./... && go test ./...`
- **Details**: Create `x/conduit/go.mod` with `module github.com/andrewhowdencom/ore/x/conduit` and `go 1.26.2`. It has no external dependencies (only `context`). Verify `x/conduit/conduit.go` and `x/conduit/conduit_test.go` compile.

### Task 2: Create `x/tool` Standalone Module
- **Goal**: Extract the shared tool interface/registry package (`x/tool/`) into its own Go module.
- **Dependencies**: None.
- **Files Affected**: `x/tool/registry.go` (reads to confirm root `provider/` import).
- **New Files**: `x/tool/go.mod`
- **Interfaces**: New module path: `github.com/andrewhowdencom/ore/x/tool`
- **Validation**: `cd x/tool && go mod tidy && go build ./... && go test ./...`
- **Details**: Create `x/tool/go.mod` with `module github.com/andrewhowdencom/ore/x/tool` and `go 1.26.2`. Add a temporary `replace github.com/andrewhowdencom/ore => ../../..` because `x/tool/registry.go` imports `github.com/andrewhowdencom/ore/provider`. `go mod tidy` will resolve `github.com/andrewhowdencom/ore` to the local root. The `replace` will be removed in Task 6 when `go.work` is introduced.

### Task 3: Delete `cmd/docgen` and Remove Generated Docs
- **Goal**: Eliminate the docgen tool (source of root→sub-module dependency) and its generated artifact.
- **Dependencies**: None.
- **Files Affected**: `cmd/docgen/main.go`, `cmd/docgen/main_test.go`
- **New Files**: None.
- **Interfaces**: None.
- **Validation**: `go build ./...` in root module still passes (cmd/ is removed, so root module no longer includes it). `go test ./...` in root passes.
- **Details**: Delete `cmd/docgen/` directory entirely. Delete `docs/conduit-capabilities.md` (it was generated by docgen). Verify no references to `cmd/docgen` remain in CI scripts, Taskfile, or Makefile.

### Task 4: Create `examples` Standalone Module
- **Goal**: Move all example applications out of the root module into a single `examples/go.mod`.
- **Dependencies**: Task 1, Task 2 (examples import `x/conduit` and `x/tool` parent packages).
- **Files Affected**: None (reads example `main.go` files to determine dependencies).
- **New Files**: `examples/go.mod`
- **Interfaces**: New module path: `github.com/andrewhowdencom/ore/examples`
- **Validation**: `cd examples && go mod tidy && go build ./...`
- **Details**: Create `examples/go.mod` with `module github.com/andrewhowdencom/ore/examples` and `go 1.26.2`. Add temporary `replace` directives for root and all sub-modules so `go mod tidy` can resolve cross-module imports locally:
  - `replace github.com/andrewhowdencom/ore => ..`
  - `replace github.com/andrewhowdencom/ore/x/conduit => ../x/conduit`
  - `replace github.com/andrewhowdencom/ore/x/tool => ../x/tool`
  - `replace github.com/andrewhowdencom/ore/x/provider/openai => ../x/provider/openai`
  - `replace github.com/andrewhowdencom/ore/x/conduit/http => ../x/conduit/http`
  - `replace github.com/andrewhowdencom/ore/x/conduit/tui => ../x/conduit/tui`
  - `replace github.com/andrewhowdencom/ore/x/tool/calculator => ../x/tool/calculator`
  - `replace github.com/andrewhowdencom/ore/x/tool/filesystem => ../x/tool/filesystem`
  After `go mod tidy`, verify all 5 example programs compile. The temporary `replace` directives will be removed in Task 6.

### Task 5: Update Root `go.mod`
- **Goal**: Remove all sub-module dependencies from the root module; add dependency on the new `x/conduit` module (since `agent/` imports it).
- **Dependencies**: Task 1, Task 3 (x/conduit module exists; cmd/docgen is gone, so root no longer needs sub-module deps).
- **Files Affected**: `go.mod`
- **New Files**: None.
- **Interfaces**: Root module no longer `require`s or `replace`s any `x/` paths.
- **Validation**: `go mod tidy && go build ./... && go test -race ./...` in root module.
- **Details**: 
  1. Remove all 6 `replace` directives for sub-modules.
  2. Remove all `require` entries for `x/conduit/http`, `x/conduit/tui`, `x/provider/openai`, `x/tool/calculator`, `x/tool/filesystem`.
  3. Add `replace github.com/andrewhowdencom/ore/x/conduit => ./x/conduit` temporarily (until Task 6 removes all replaces).
  4. Run `go mod tidy`. Verify `go build ./...` and `go test -race ./...` pass.

### Task 6: Update All Sub-Module `go.mod` Files
- **Goal**: Remove `replace` directives for root; add explicit dependencies on `x/conduit` or `x/tool` where sub-modules import them.
- **Dependencies**: Task 1, Task 2, Task 5.
- **Files Affected**: 
  - `x/provider/openai/go.mod`
  - `x/conduit/http/go.mod`
  - `x/conduit/slack/go.mod`
  - `x/conduit/telegram/go.mod`
  - `x/conduit/tui/go.mod`
  - `x/tool/bash/go.mod`
  - `x/tool/calculator/go.mod`
  - `x/tool/filesystem/go.mod`
  - `x/tool/mcp/go.mod`
- **New Files**: None.
- **Interfaces**: Sub-modules reference root by pseudo-version; conduit sub-modules reference `x/conduit` module; tool sub-modules reference `x/tool` module.
- **Validation**: `go mod tidy && go build ./... && go test ./...` in each sub-module.
- **Details**: For each sub-module `go.mod`:
  1. Remove `replace github.com/andrewhowdencom/ore => ../../..`.
  2. Change `require github.com/andrewhowdencom/ore v0.0.0` to `require github.com/andrewhowdencom/ore v0.0.0-00010101000000-000000000000` (or let `go mod tidy` resolve it).
  3. For conduit sub-modules (`x/conduit/*`): add `require github.com/andrewhowdencom/ore/x/conduit v0.0.0` and temporary `replace github.com/andrewhowdencom/ore/x/conduit => ../../conduit`.
  4. For tool sub-modules (`x/tool/*`): add `require github.com/andrewhowdencom/ore/x/tool v0.0.0` and temporary `replace github.com/andrewhowdencom/ore/x/tool => ../..` (or adjust path: `../../tool` from `x/tool/bash`).
  5. Run `go mod tidy` in each sub-module.
  6. Verify `go build ./...` and `go test ./...` pass in each.

### Task 7: Create `go.work` and Remove All `replace` Directives
- **Goal**: Enable workspace-based local development and eliminate all `replace` directives from published `go.mod` files.
- **Dependencies**: Task 4, Task 5, Task 6 (all modules must exist and have correct dependencies before workspace sync).
- **Files Affected**: `go.mod`, `examples/go.mod`, `x/conduit/go.mod`, `x/tool/go.mod`, all 9 sub-module `go.mod` files
- **New Files**: `go.work`, `.gitignore` (to add `go.work.sum`)
- **Interfaces**: Workspace lists all 13 modules.
- **Validation**: `go work sync && go build ./... && go test -race ./...` from repository root.
- **Details**:
  1. Create `go.work` at repo root:
     ```
     go 1.26.2
     use (
         .
         ./examples
         ./x/conduit
         ./x/conduit/http
         ./x/conduit/slack
         ./x/conduit/telegram
         ./x/conduit/tui
         ./x/provider/openai
         ./x/tool
         ./x/tool/bash
         ./x/tool/calculator
         ./x/tool/filesystem
         ./x/tool/mcp
     )
     ```
  2. Remove ALL `replace` directives from every `go.mod` file:
     - Root `go.mod`: remove `replace github.com/andrewhowdencom/ore/x/conduit => ./x/conduit`
     - `examples/go.mod`: remove all replaces
     - `x/tool/go.mod`: remove `replace github.com/andrewhowdencom/ore => ../../..`
     - All sub-module `go.mod`s: remove their temporary replaces for `x/conduit`, `x/tool`, and root
  3. Run `go work sync`. This populates workspace `go.sum` and resolves cross-module references.
  4. Run `go mod tidy` in each module to clean up.
  5. Run `go build ./...` from root (workspace mode builds all modules).
  6. Run `go test -race ./...` from root.
  7. Add `go.work.sum` to `.gitignore` (do NOT add `go.work` itself; it should be committed for shared workspace config).

### Task 8: Verify External-Consumer Path
- **Goal**: Confirm that a fresh module can `go get` a sub-module without errors.
- **Dependencies**: Task 7.
- **Files Affected**: None.
- **New Files**: None.
- **Interfaces**: None.
- **Validation**: Simulated `go get` succeeds.
- **Details**: Create a temporary directory outside the repo. Initialize `go mod init testmod`. Run `go get github.com/andrewhowdencom/ore/x/provider/openai@latest` using a local `GOPROXY=file://...` or verify that `go list -m -versions github.com/andrewhowdencom/ore/x/provider/openai` works. Alternatively, since modules aren't published yet, verify with `go mod graph` that no `replace` directives remain and that the module graph is acyclic. Check that `go list -m all` in each module shows no `replace` entries.

## Risks & Mitigations

| Risk | Impact | Likelihood | Mitigation |
|---|---|---|---|
| `go work sync` fails due to unresolved cross-module references after `replace` removal | High | Medium | Keep `replace` directives in place during Tasks 1–6; remove them only in Task 7 after `go.work` is created. If sync fails, re-add targeted `replace` directives temporarily and debug the specific module. |
| Sub-module `go.mod` paths for `x/tool` replace are incorrect (e.g., `../../tool` vs `../..`) | Medium | Medium | Double-check relative paths: from `x/tool/bash`, `x/tool` is at `../..`. From `x/conduit/http`, `x/conduit` is at `../../conduit`. Validate with `go mod tidy` before removing replaces. |
| Example programs fail to compile after module boundary changes | Medium | Low | `go build ./...` in `examples/` during Task 4 validates all 5 programs. Fix import paths or missing `replace` directives immediately. |
| Root module tests fail due to missing `x/conduit` dependency | Medium | Low | Task 5 adds `x/conduit` to root. `agent/agent.go` and `agent/agent_test.go` are the only root packages importing `x/conduit`. Verify with `go test ./agent/...` during Task 5. |
| `go.work` introduces unexpected workspace behavior (e.g., `go test ./...` from root skips non-workspace modules) | Low | Low | `go.work` `use` directives explicitly list all modules. `go build ./...` from root builds the active module (root). Use `go work sync` and test individual modules with `cd <module> && go test ./...`. |
| `cmd/docgen` is referenced in CI or build scripts | Medium | Low | Search `Taskfile.yml`, `.github/workflows/`, `Makefile` for `docgen` references during Task 3. Remove or update any found. |
| `docs/conduit-capabilities.md` is linked from README or other docs | Low | Low | Check `README.md` and `docs/` for links to `conduit-capabilities.md`. If found, update link or note the file is now manually maintained. |
