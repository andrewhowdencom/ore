# Plan: Implement Structured Handoff Summary Prompt (Issue #384)

## Objective

Redesign the `SummarizeStrategy` default prompt to produce a structured, deterministic, two-pass handoff summary with markdown-formatted sections. Add a `Prompt` field to `SummarizeStrategy` so applications can override the default. The detailed rationale and design discussion are captured in `.plans/structured-handoff-summary-prompt.md`.

## Context

- `x/compaction/summarize.go` contains `SummarizeStrategy` with a hardcoded generic prompt: `"Summarize the above conversation concisely, preserving all key facts, decisions, and context."`
- `x/compaction/summarize_test.go` contains a comprehensive test suite with a `mockProvider` that captures `receivedTurns` for assertion.
- `x/compaction/doc.go` documents the package API including `SummarizeStrategy`.
- No external consumers of `SummarizeStrategy` exist outside `x/compaction/` (verified by `grep`).
- The existing test suite passes and must continue to pass without modification (the mock provider does not depend on prompt content).
- The package follows a **plain-struct pattern** — no constructors, no functional options. Adding a field to `SummarizeStrategy` is consistent with `KeepLastN`, `TurnCountTrigger`, and `TokenUsageTrigger`.

## Architectural Blueprint

Add a `Prompt string` field to `SummarizeStrategy`. When `Prompt` is empty, `Compact` uses an unexported package-level `defaultPrompt` constant containing the structured handoff prompt. When `Prompt` is non-empty, it uses the application-provided prompt verbatim. This is additive and breaks no existing API.

## Requirements

1. `SummarizeStrategy` gains a `Prompt string` field.
2. `Compact` uses `s.Prompt` when non-empty, otherwise falls back to `defaultPrompt`.
3. The `defaultPrompt` is an unexported constant with a structured handoff prompt that instructs the model to internally analyze, then output ONLY a markdown summary with exactly five sections: Primary Goal, Key Decisions & Constraints, Completed Work, Current State / Work in Progress, Pending Tasks & Next Steps.
4. All existing tests in `x/compaction/summarize_test.go` continue to pass unchanged.
5. New tests verify default prompt usage and custom prompt override behavior.
6. `x/compaction/doc.go` documents the `Prompt` field and the default prompt behavior.

### Task 1: Add `defaultPrompt` constant and `Prompt` field to `SummarizeStrategy`
- **Goal**: Add the unexported `defaultPrompt` constant and the `Prompt` field to `SummarizeStrategy`, and wire the field into `Compact`.
- **Dependencies**: None.
- **Files Affected**: `x/compaction/summarize.go`
- **New Files**: None.
- **Interfaces**:
  - `SummarizeStrategy` struct: add `Prompt string` field.
  - `Compact` method: replace hardcoded prompt string with `prompt := s.Prompt; if prompt == "" { prompt = defaultPrompt }`.
- **Validation**:
  - `go build ./x/compaction/...` compiles.
  - `go test ./x/compaction/...` passes (all existing tests must pass unchanged).
- **Details**:
  - Add the following unexported constant at the top of `x/compaction/summarize.go` (after the `estimateTokens` function is fine, or before the struct — any package-level location works):
    ```go
    const defaultPrompt = `You are creating a context handoff summary for another agent that will resume this conversation.

First, analyze the conversation history to identify:
- The primary goal or task the user is trying to accomplish
- Key decisions, constraints, preferences, and rules established
- Completed work and successful outcomes
- Current state and work in progress
- Pending tasks and next steps required

Then, output ONLY a structured summary using the following markdown sections. Do not include any analysis, reasoning, introductory text, or commentary outside these sections. The summary must be concise but complete — another agent will resume work from this summary.

## Primary Goal
[What the user is trying to accomplish]

## Key Decisions & Constraints
[Important rules, preferences, architectural choices, or constraints established]

## Completed Work
[What has already been successfully done]

## Current State / Work in Progress
[What is actively being worked on, including any in-progress decisions or partial results]

## Pending Tasks & Next Steps
[What still needs to be done to complete the goal]`
    ```
  - Update `SummarizeStrategy` struct:
    ```go
    type SummarizeStrategy struct {
        Provider  provider.Provider
        MaxTokens int
        Prompt    string
    }
    ```
    Add a doc comment for the new field: `// Prompt is an optional custom summarization prompt. When empty, a default structured handoff prompt is used.`
  - In `Compact`, replace:
    ```go
    buf.Append(state.RoleUser, artifact.Text{
        Content: "Summarize the above conversation concisely, preserving all key facts, decisions, and context.",
    })
    ```
    with:
    ```go
    prompt := s.Prompt
    if prompt == "" {
        prompt = defaultPrompt
    }
    buf.Append(state.RoleUser, artifact.Text{Content: prompt})
    ```
  - Ensure the rest of `Compact` — control flow, error handling, `RoleSystem` assignment, `toSummarize`/`toPreserve` logic — is completely unchanged.

- **Dependencies**: Task 1.

- **Interfaces**: None.
- **Validation**:

- **Details**:
  - Add `TestSummarizeStrategy_UsesDefaultPrompt`:
    - Create `SummarizeStrategy{Provider: prov, MaxTokens: 1, Prompt: ""}`.
    - Use two turns of 1 token each (e.g., `textTurn(state.RoleUser, "aaaa")` twice) to trigger summarization.
    - Configure the mock provider to return `[]artifact.Artifact{artifact.Text{Content: "Summary."}}`.
    - After `Compact`, assert that `prov.called` is true.
    - Assert that `prov.receivedTurns` has length 3 (2 summarized turns + 1 appended user prompt turn).
    - Assert that the last turn's role is `state.RoleUser` and its only artifact is `artifact.Text` with content equal to `defaultPrompt`.
    - Since `defaultPrompt` is unexported, use `assert.Equal(t, defaultPrompt, text.Content)` from within the same package.
  - Add `TestSummarizeStrategy_UsesCustomPrompt`:
    - Same setup but with `Prompt: "Custom override prompt"`.
    - Assert that the last user turn's text content equals `"Custom override prompt"`.
  - Both tests should be table-driven if sharing setup logic, or separate `func` declarations if simpler.

- **Details**:
  - In the `# Built-in Strategies` section, update the `SummarizeStrategy` paragraph to:
    ```
    SummarizeStrategy is a token-aware strategy that calls an LLM provider to
    summarize conversation history. It walks backwards from the last turn,
    preserving the suffix that fits within MaxTokens, and replaces the prefix
    with a single synthetic system summary turn. If the entire history fits,
    it returns a defensive copy without calling the provider. Token estimation
    is a rough heuristic (len(text)/4) with no external dependencies.
    The summary turn uses RoleSystem because it is injected context about
    prior conversation, not a real assistant response.

    SummarizeStrategy uses a default structured handoff prompt that produces
    markdown output with five sections: Primary Goal, Key Decisions &
    Constraints, Completed Work, Current State / Work in Progress, and Pending
    Tasks & Next Steps. Applications can override the prompt via the Prompt
    field.
    ```

## Risks & Mitigations

| Risk | Impact | Likelihood | Mitigation |
|---|---|---|---|
| Default prompt is too long, increasing token cost | Low | Medium | Prompt is ~1.5KB; negligible compared to conversation history. Quality justifies cost. |
| Default prompt does not work well with all providers (e.g. local models) | Medium | Low | Prompt uses simple, explicit instructions and standard markdown. The `Prompt` field allows substitution. |
| Existing external consumers depend on the hardcoded prompt string | Low | High | No external consumers in repo. The hardcoded string was never exported. The `Prompt` field is additive. |
| Multi-line raw string literal in Go source is tricky to format | Low | Low | Use backticks. Ensure no leading/trailing blank lines that would affect prompt content. |
