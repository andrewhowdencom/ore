# Plan: x/provider/anthropic (Anthropic Messages API + OpenRouter /messages mirror)

## Objective

Close issue #433 by delivering a fully working `x/provider/anthropic` provider package that:
1. Implements the Anthropic Messages API streaming protocol end-to-end, including reasoning (`thinking`, `redacted_thinking`) round-trip across turns.
2. Is wire-compatible with the OpenRouter `/api/v1/messages` mirror, selecting the right auth header per host.
3. Surfaces the provider to ore applications via the standard `provider.Provider` interface so it can be composed in any application that today uses `x/provider/openai`.

## Architectural Blueprint

**Selected architecture: extend the WIP branch's SDK-based provider, complete the streaming `Invoke`, and host-aware auth dispatch.**

The WIP chose to wrap the official `github.com/anthropics/anthropic-sdk-go` v1.50.1. The SDK exposes `MessageService.NewStreaming(ctx, params, opts...)` returning a `ssestream.Stream[MessageStreamEventUnion]`, where the union has `AsMessageStart`, `AsMessageDelta`, `AsMessageStop`, `AsContentBlockStart`, `AsContentBlockDelta`, `AsContentBlockStop` accessors and the delta union (`RawContentBlockDeltaUnion`) has `AsThinkingDelta`, `AsTextDelta`, `AsInputJSONDelta`, `AsSignatureDelta`, `AsCitationsDelta`. The SDK automatically sets `anthropic-version: 2023-06-01` (verified in `internal/requestconfig/requestconfig.go:168`).

The remaining work builds on top of that. After merging the WIP, the plan is:

1. **Task 0 — update `AGENTS.md`**: relax the "Dependencies" subsection to acknowledge the per-extension go.mod convention and that external SDKs are acceptable under isolation. This is documentation only.
2. **Task 1 — merge the WIP branch** into the current `433`. Expect a clean merge since the WIP branched from `d520f6f`, which is HEAD on `433`.
3. **Task 2 — host-aware auth dispatch**: extend `buildSDKOptions` to pick `option.WithAuthToken` (for `Authorization: Bearer`) when `isOpenRouter(cfg.baseURL)`, vs `option.WithAPIKey` (for `x-api-key`) for Anthropic native. Cover with `TestNew_OpenRouterBaseURL` and `TestNew_AnthropicNativeBaseURL` by inspecting the recorded request headers via the mock `http.RoundTripper`.
4. **Task 3 — streaming `Invoke`**: replace the stub with a real implementation that walks the SDK's event stream, dispatches each event variant to the right ore artifact, and emits on the channel in arrival order. The dispatch table is:
   - `MessageStartEvent` → no-op (envelope).
   - `ContentBlockStartEvent{ThinkingBlock}` → record block index (no artifact).
   - `ContentBlockStartEvent{RedactedThinkingBlock}` → emit `ReasoningSignature{Provider: "anthropic", SubKind: "redacted", Data: block.Data}`.
   - `ContentBlockStartEvent{TextBlock}` → record block index (no artifact).
   - `ContentBlockStartEvent{ToolUseBlock}` → emit `ToolCallDelta{Index: event.Index, ID: block.ID, Name: block.Name}`.
   - `ContentBlockDeltaEvent{ThinkingDelta}` → emit `ReasoningDelta{Content: delta.Thinking}`.
   - `ContentBlockDeltaEvent{SignatureDelta}` → emit `ReasoningSignature{Provider: "anthropic", SubKind: "signature", Data: delta.Signature}`.
   - `ContentBlockDeltaEvent{TextDelta}` → emit `TextDelta{Content: delta.Text}`.
   - `ContentBlockDeltaEvent{InputJSONDelta}` → emit `ToolCallDelta{Index: event.Index, Arguments: delta.PartialJSON}`.
   - `ContentBlockDeltaEvent{CitationsDelta}` → no-op (defensive; not in the issue but the SDK models it).
   - `ContentBlockStopEvent` → no-op.
   - `MessageDeltaEvent` → buffer the usage; emit a single `Usage` artifact at `MessageStopEvent` with all known fields including `ThinkingTokens: event.Usage.OutputTokensDetails.ThinkingTokens`. Emit-once at stop avoids the double-count trap (the SDK's `MessageDeltaUsage` is cumulative, so emitting per-`MessageDeltaEvent` would multiply-count tokens).
   - `MessageStopEvent` → break.
   - Unknown variants fall through silently.
5. **Task 4 — OpenTelemetry integration**: open a `provider.invoke` span (kind=client) when `p.tracer != nil`. Set `model` and `thread_id` attributes (using `loop.ThreadIDFrom(ctx)`). Attach `httptrace.WithClientTrace(ctx, otelhttptrace.NewClientTrace(ctx, otelhttptrace.WithoutSubSpans()))` so the span records DNS / connect / TLS / first-byte lifecycle events without child sub-spans. On error, `span.RecordError(err)` and `span.SetStatus(codes.Error, ...)`. Defer `span.End()`.
6. **Task 5 — streaming tests**: the five tests the issue's acceptance criteria names, plus context-cancellation and tool-use streaming. Mock transport pattern: a `recordingMockTransport` similar to the one in `x/provider/openai/openai_test.go` (mutex-protected, captures request, returns a canned SSE response body). The SDK accepts a `*http.Client` via `option.WithHTTPClient`, so the mock transport plugs in directly.
7. **Task 6 — verify**: `task x-provider-anthropic:validate` and the root `task validate` must both pass.

**Tree-of-Thought deliberation (recorded):**

| Path | Verdict | Why |
|---|---|---|
| Hand-rolled `net/http` + SSE parser (AGENTS.md's "stdlib only" rule) | Rejected | AGENTS.md's rule is being relaxed in Task 0; the WIP already chose the SDK; pivoting now would discard ~1100 LoC of WIP code and tests. The streaming protocol is small but the wire schema (auth, version, request body) has a long tail. |
| WIP's SDK-based approach | Selected | Consistent with the openai module's `openai-go` choice. Same per-extension go.mod isolation. Less code, fewer wire-protocol edge cases to test. |
| `loop.Step` integration: provider emits the final accumulated `Reasoning` with signature at end of turn (replaces the loop-accumulated one) | Rejected | The loop's pipeline does not support replacement; complete artifacts flush the accumulated deltas and are appended after, producing duplicates. The signature travels as a separate `ReasoningSignature` artifact, which the WIP's write-side already handles by attaching to the preceding thinking block. |
| Emit `ReasoningSignature` as a `ReasoningSignatureDelta` (delta type, merges into the in-progress `Reasoning` accumulator slot) | Rejected | Requires extending the artifact package's delta contract and the loop's accumulator. The WIP chose the cleaner separation: signatures are complete artifacts emitted at the close of a reasoning block, not deltas. |

## Requirements

1. **Streaming parity with the openai module**: the anthropic provider MUST emit `ReasoningDelta`, `TextDelta`, `ToolCallDelta`, `ReasoningSignature`, and `Usage` artifacts on the channel in wire-arrival order, with `select { case ch <- ...: case <-ctx.Done(): return ctx.Err() }` guarding every send.
2. **Reasoning round-trip**: prior-turn `Reasoning` and `ReasoningSignature{anthropic, ...}` artifacts MUST be re-serialized as `thinking` blocks (signature attached in place) and `redacted_thinking` blocks on the next request. The WIP's `serializeMessages` already implements this; the streaming work must not break it.
3. **Usage with thinking tokens**: the `Usage` artifact emitted at `MessageStopEvent` MUST populate `ThinkingTokens` from `event.Usage.OutputTokensDetails.ThinkingTokens` when the host reports it, and MUST leave it zero (and therefore omitted from the JSON payload via `omitempty`) otherwise. `[inferred]` — the issue names the field as `usage.output_tokens_details.thinking_tokens`; the artifact's `omitempty` tag is the way to make "no thinking reported" indistinguishable from "zero thinking" on the wire.
4. **Host-aware auth**: the auth header MUST be selected at construction from the base URL (`isOpenRouter` heuristic). Anthropic native → `x-api-key` (via `option.WithAPIKey`). OpenRouter → `Authorization: Bearer <key>` (via `option.WithAuthToken`). The decision MUST NOT be re-walked per invocation.
5. **OpenTelemetry span**: when `WithTracer` is configured, `Invoke` MUST open a `provider.invoke` span (kind=client), record the `model` (effective, post-`provider.WithModel` override) and `thread_id` (via `loop.ThreadIDFrom`) attributes, attach the `otelhttptrace` client trace, and end the span on return.
6. **Companion openai changes**: the read-side and write-side changes to `x/provider/openai` (delta.reasoning + delta.reasoning_details[] read, reasoning replay write) are part of the WIP merge and MUST land in the same commit series.
7. **Workspace integration**: the new module MUST be listed in `go.work` and `Taskfile.yml`'s `validate` task. The WIP already does this; the merge carries it.
8. **AGENTS.md update**: the "Dependencies" section MUST be rewritten to reflect the per-extension go.mod convention and the acceptance of external SDKs under isolation.

### Task 3: Implement streaming `Invoke`

- **Goal**: Replace the stub `Invoke` with a real streaming implementation that maps SDK events to ore artifacts.
- **Dependencies**: Task 1, Task 2.
- **Files Affected**: `x/provider/anthropic/anthropic.go` (the `Invoke` method).
- **New Files**: None.
- **Interfaces**: No new exports. The signature of `Invoke` is unchanged: `func (p *Provider) Invoke(ctx context.Context, s state.State, ch chan<- artifact.Artifact, opts ...provider.InvokeOption) error`. Internal helpers may be added (e.g., `dispatchEvent`); these are unexported.
- **Validation**:
  - `TestInvoke_StubReturnsNil` is repurposed or deleted (it only asserted the stub returns nil, which is no longer the desired behavior). Recommend: delete it; the new streaming tests cover the live path. If the user prefers to keep it, repurpose it to test a minimal-state error path (e.g., `ctx` canceled before the call).
  - `task x-provider-anthropic:validate` passes after Task 5's tests are in place.
- **Details**:
  1. **Build the request.** Construct `anthropic.MessageNewParams{Model: anthropic.Model(p.model), Messages: serialized.Messages, MaxTokens: ...}`. Set `System` to the serialized system field. Set `Temperature`, `MaxTokens`, `Tools` from `applyInvokeOptions(opts)`. Set `Thinking` via `anthropic.ThinkingConfigParamOfEnabled(budget)` only if `thinkingBudgetSet` is true and the budget is non-zero.
  2. **Honor the per-invocation model override.** If a `provider.ModelOption{Model: ...}` is in `opts`, override `Model` accordingly. Precedence: per-invocation `provider.WithModel(...)` > constructor `WithModel(...)` (the loop's `Stream.ModelOption` is the documented call site for the override). Empty `Model` is a no-op (per the contract documented in `provider/provider.go`).
  3. **Open the OTel span** (Task 4 details). Defer `span.End()`.
  4. **Attach the client trace** (Task 4 details).
  5. **Call `p.client.Messages.NewStreaming(ctx, params, sdkOpts...)`.** Note: `sdkOpts` is a closed set captured at construction. The current provider does not store the resolved `sdkOpts` on the struct; either store it on the `Provider` struct (simplest), or rebuild it from the stored fields. Storing it is cleaner. The merge will not break either way; this task makes the choice explicit.
  6. **Walk the event stream:**
     ```go
     stream := p.client.Messages.NewStreaming(ctx, params, sdkOpts...)
     defer stream.Close()
     var pendingUsage *artifact.Usage
     for stream.Next() {
         event := stream.Current()
         if err := p.dispatchEvent(event, ch, &pendingUsage); err != nil {
             return err
         }
         if _, ok := event.AsMessageStop(); ok {
             break
         }
     }
     if err := stream.Err(); err != nil {
         return fmt.Errorf("anthropic streaming: %w", err)
     }
     if pendingUsage != nil {
         select {
         case ch <- *pendingUsage:
         case <-ctx.Done():
             return ctx.Err()
         }
     }
     return nil
     ```
  7. **`dispatchEvent` switch.** See the dispatch table in the "Architectural Blueprint" section above. For each branch, send on `ch` with `select { case ch <- art: case <-ctx.Done(): return ctx.Err() }`. Skip empty deltas (`Content == ""`, `Arguments == ""`, `Data == ""`).
  8. **Error propagation.** If the SDK returns a `MessageStreamEventUnion` whose `AsAny()` returns an error variant (rare; the SDK typically returns errors via `stream.Err()`), wrap with `fmt.Errorf("anthropic stream: %w", err)`.

| Risk | Impact | Likelihood | Mitigation |

| WIP merge conflicts with the current `main` | High | Low | The WIP branched from `d520f6f`, which is HEAD on `433`. See Task 3, step 6, and the dispatch table in the Architectural Blueprint. |
